## 🚨 문제 정의

가짜 뉴스뿐만 아니라 **특정 정치적 입장에 치우친 편향적인 뉴스**의 범람으로 인해 언론에 대한 불신과 확증 편향(필터 버블)이 심각한 사회적 문제로 대두되고 있습니다. 독자들은 하나의 기사만 읽고 사실을 오인하거나 치우친 시각을 갖기 쉬우며, 객관적이고 다각적인 시각에서 사건을 바라볼 수 있는 보조 도구가 절대적으로 부족한 상황입니다.

---

## 👥 대상 사용자

뉴스 기사를 소비하면서 정치적 편향에서 벗어나 균형 잡힌 시각을 유지하고자 하는 **일반 대중** 및 **미디어 리터러시 교육**에 관심이 높은 독자층.

---

## ⭐ 핵심 기능 (AI & Data)

1. **실시간 트렌딩 뉴스 수집 & 시맨틱 중복 제거**: 네이버 뉴스 API를 통해 실시간 뉴스를 고속 병렬 수집하며, 기사 제목의 형태소를 분석해 **의미가 중복되는 기사를 지능적으로 필터링**합니다.
2. **초경량 머신러닝 편향성 예측 엔진**: 26개 주요 언론사의 대량 기사를 사전에 학습한 **245KB 초경량 분류기(Logistic Regression + TF-IDF + LDA)**를 통해 기사별 진보·중도·보수 성향 비율(%)을 1ms 내에 도출합니다.
3. **Gemini 다각적 리포트 & 대체 헤드라인**: Gemini 2.5 Flash를 활용해 편향 포인트와 팩트체크를 짚어내고, 동일한 사건을 **중립/진보/보수 3가지 프레임의 헤드라인으로 자동 재작성**하여 보여줍니다.
4. **GraphRAG 프레임 지식 그래프 시각화**: 기사 본문 속 인물/사건/개념의 인과 관계와 대립 구도를 LLM이 추출하여, 직관적인 **네트워크 지식 그래프(Knowledge Graph)**로 그려냅니다.
5. **실시간 대화형 AI 패널**: 기사를 읽다가 생긴 궁금증이나 숨겨진 의도 등을 실시간으로 AI 챗봇에게 묻고 답변받을 수 있습니다.

---

## 🧠 입력 → AI 처리 → 출력 (Pipeline)

1. **사용자 액션**: 실시간 뉴스 클릭 또는 기사 URL 직접 입력.
2. **백엔드 고속 스크래핑**: Connection Pooling 및 Multi-threading을 통해 기사 본문과 사진(`og:image`)을 2초 내외로 추출 (Fallback 파서 적용).
3. **로컬 머신러닝 (ML)**: 추출된 텍스트를 `bias_model.pkl`에 통과시켜 정치 성향 3단계 비율(%) 계산.
4. **대규모 언어 모델 (LLM)**: ML 결과를 프롬프트에 결합하여 Gemini API 호출 → 편향성 논평, 팩트체크, 대체 헤드라인, GraphRAG 노드 추출 수행.
5. **프론트엔드 출력**: 대시보드 화면에 다이내믹 UI(Glassmorphism), 막대 그래프, 네트워크 시각화 렌더링 및 챗봇 인터페이스 활성화.

---

## ⚙️ 정치 편향성 머신러닝 분류기 학습 구조 (상세)

1. **데이터셋 (Dataset)**: 대한민국 주요 26개 언론사가 보도한 정치/사회 기사 텍스트 약 20,000건 크롤링 구축.
2. **텍스트 정제 & 특징 추출**: HTML 태그 및 노이즈 제거 후, **TF-IDF (단어 빈도 가중치)** 와 **LDA (잠재 디리클레 할당 토픽 모델링)** 를 결합해 문맥의 핵심 어휘 패턴 벡터 생성.
3. **분류 알고리즘 (Algorithm)**: **다중 클래스 로지스틱 회귀(Logistic Regression)**에 L2 규제(Ridge)를 튜닝 적용하여 과적합 방지.
4. **모델 경량화 (Optimization)**: 전체 파이프라인을 `joblib`로 병합 압축하여 **245KB** 단일 파일로 배포. 무거운 딥러닝 없이도 높은 정확도와 초고속 응답 달성.

## 🛠️ 기술 스택

- **UI**: FastAPI + HTML5/CSS3 (Glassmorphism) + Vanilla JS + `vis-network` (그래프)
- **Model / API**: Gemini 2.5 Flash API, scikit-learn (LogisticRegression, TfidfVectorizer, LDA)
- **Scraping / Backend**: `requests.Session` (Connection Pooling), `concurrent.futures`, `BeautifulSoup4`
- **Infrastructure**: AWS EC2 + Docker Compose + Nginx (Reverse Proxy)
