가짜 뉴스를 넘어 편향을 분석하다

가짜 뉴스 뿐만이 아니라 편향된 뉴스로 현재 대한민국은 언론의 도마 위에 있습니다. 이러한 현실에 입각하여 본 프로젝트는 독자들이 특정 진영의 생각에 갇히는 필터 버블(Filter Bubble) 현상을 깨뜨리고자 기획되었습니다.

저희 '뉴스밸런스 (NewsBalance)'는 주요 신문사를 통해 수집된 뉴스를 대상으로 작동하며, 무조건적인 가짜 뉴스 배제와 더불어 하나의 시사 이슈에 대해 다각적인 관점을 비교하여 볼 수 있는 기회를 선사합니다.

모든 수집 기사에 대해서는 사전 학습을 통하여 분리한 어휘 및 표현적 기준 데이터를 토대로 머신러닝 분류 알고리즘이 동작하며, 해당 기사가 진보·중도·보수 중 어느 성향에 가까운지를 분석하여 직관적인 백분율(%)로 표기합니다.

원하는 뉴스를 클릭하면 단순 수치 분석을 넘어, 편향성 완화를 위해 해당 뉴스를 균형 잡힌 시각에서 바라볼 수 있도록 보조하는 AI(Gemini) 종합 리포트 및 질의응답 챗봇 시스템을 함께 가동합니다.

본 프로젝트는 차후 한국을 넘어 해외 각국으로 대상을 확장하여, 각 국가별 고유한 자국 시선으로도 글로벌 뉴스를 교차 감상할 수 있는 다각화된 뷰를 제시하고자 합니다. 독자 여러분이 이 거대한 세계 속에서 편향되지 않은 넓고 올바른 시야를 발견할 수 있기를 기대합니다.

Technical Composition & System Architecture

1. 기술 스택 & 시스템 설계

  • 프론트엔드/백엔드: FastAPI (Uvicorn) 비동기 백엔드 서버 + 반응형 HTML5/CSS3 (Glassmorphism 다크 테마) 및 Vanilla JS 구현.
  • 머신러닝 분류기: 26개 주요 언론사 기사 데이터를 학습한 Logistic Regression 분류기 탑재. TF-IDF 어휘 특징과 LDA 토픽 모델링 데이터를 기반으로 기사 성향을 실시간 백분율(%)화. 모델 크기 245KB (`bias_model.pkl`).
  • 생성 AI 모델: `google-genai` 최신 라이브러리로 Gemini 2.5 Flash API 연동. 편향성 경고, 타 언론사 비교, 팩트체크 리포트 작성 및 질의응답 챗봇 기능 수행.
  • 외부 API 연동: 네이버 뉴스 OpenAPI를 통하여 시사/정치/국제 카테고리별 실시간 뉴스를 주기적으로 수집 및 파싱.

2. 배포 및 완료 조건

  • 완료 조건: 뉴스 리스트의 기사를 클릭했을 때, 10초 이내에 기사의 진보/중도/보수 분석 그래프가 나타나고 Gemini의 4대 편향 종합 보고서 및 AI 질의응답 기능이 화면에 지연 없이 렌더링된다.
  • 클라우드 인프라: AWS EC2 (Ubuntu 22.04 LTS) 가상 서버 환경에 탄력적 IP(Elastic IP)를 매핑하고, Docker Compose를 사용하여 FastAPI 컨테이너와 Nginx 웹 프록시 서버를 기동.
  • CI/CD 자동화: GitHub Actions 워크플로우를 활용해 코드 push 시 SSH 원격 접속 후 자동으로 Nginx와 앱 서비스를 무중단 빌드 및 재실행.
  • n8n 워크플로우 서버: 동일한 AWS EC2 서버 환경 위에 Docker Compose 및 Caddy를 묶어 12주차 n8n 자동화 인프라 서버를 병행 가동.

3. 머신러닝 분류 모델 설계 & 학습 파이프라인

  • 데이터셋 수집 및 군집화: 26개 주요 언론사가 생산한 약 2만 건의 실제 정치/사회 분야 뉴스 본문 텍스트 데이터를 웹 크롤링 및 네이버 OpenAPI를 통해 사전에 수집하여 정밀 학습 말뭉치(Corpus)를 구축했습니다.
  • 텍스트 전처리 및 어휘 특징 추출 (TF-IDF): 텍스트 내 HTML 태그, 불필요한 조사와 공백 등을 정규표현식으로 정밀 필터링한 후, 단어별 상대적 중요도와 문맥 상의 단어 조합 흐름을 모두 학습하기 위해 Uni-gram과 Bi-gram을 병합한 TF-IDF (Term Frequency-Inverse Document Frequency) 벡터화를 적용했습니다.
  • 토픽 모델링 특징 결합 (LDA): 단순한 어휘 빈도를 넘어 기사가 전반적으로 내포하고 있는 주제를 파악하기 위해 LDA (Latent Dirichlet Allocation) 알고리즘을 도입, 각 기사가 가질 수 있는 다차원 토픽 확률 벡터를 별도 특징으로 결합했습니다.
  • 정치 성향 로지스틱 회귀 학습 (Logistic Regression): 추출된 복합 특징 데이터들을 다중 클래스(Multi-class) 로지스틱 회귀 분류기 모델에 매핑하여 학습하였습니다. L2 규제(Ridge Penalty)를 적용하여 데이터의 과적합(Overfitting)을 차단했습니다.
  • 모델 경량화 및 고성능 직렬화: 전체 학습 파이프라인을 245KB 크기의 단일 가벼운 모델 파일(`bias_model.pkl`)로 경량화 직렬화하여, AWS EC2와 같은 제한된 클라우드 환경에서도 추론 호출 시 1ms 미만의 극도로 민첩한 실시간 예측(Inference)을 가능하게 했습니다.
